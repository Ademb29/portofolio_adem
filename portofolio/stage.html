L'entreprise

La Total Energies Digital Factory (TDF) est le centre d'excellence technologique du groupe Total Energies, l'un des leaders mondiaux de l'énergie. Structurée comme une véritable "usine numérique" interne, elle regroupe plus de 300 experts — développeurs, Data Scientists, Product Owners — organisés en équipes autonomes appelées Squads.

La TDF travaille en partenariat stratégique avec la société indienne TCS (Tata Consultancy Services), rendant l'anglais obligatoire dans toutes les réunions. L'organisation est rythmée par des cycles de 4 mois, des semaines de Scoping (planification) et des Runs de 2 semaines pour l'exécution.

Partie 1 — Générateur de données (Équipe Naofel)

Le projet WOE mesure les performances de sites offshore pétroliers dans le monde (Nigeria, Brésil, Qatar…). Pour tester les outils sans utiliser de données réelles ni mettre les sites en maintenance, ma mission était de créer un générateur de données fictives.

1

Générateur Python/Flask — Version initiale

Bouton unique → génération instantanée de données simulées

Développement d'une classe principale NumValue avec des classes filles spécifiques (Pressure, Current…). Routes Flask GET (affichage) et POST (génération). Affichage des valeurs simulées (ampères, bars, hertz) directement sur la page.

Python Flask POO Routes GET/POST
2

Améliorations — Saisie dynamique & chronologie

Date de début, date de fin, nombre de données par jour

Ajout de 3 champs de saisie (date début, date fin, quantité/jour). Boucle calculant l'écart entre les deux dates pour générer le volume exact. Refactorisation avec une classe Data dédiée. Affichage chronologique avec heure, minute, seconde — rendu proche d'un flux industriel réel.

Algorithmique Refactorisation UX
3

Connexion base de données

Persistance des données générées en table SQL

Création d'une base de données et connexion au site Flask. Les données générées s'affichent à l'écran ET sont insérées dans une table data en base. Permet à l'équipe de rejouer des scénarios de test complets sans toucher aux données réelles.

SQL Flask-SQLAlchemy Persistance
La semaine de Scoping entre les deux parties a été une découverte : suspendre le développement pour réfléchir collectivement à la définition du succès, analyser les données capteurs avec l'équipe Data Science, et planifier le prochain cycle via la méthode MVE (Minimum Viable Ecosystem). Une vision globale du cycle de vie d'un produit que l'IUT ne m'avait pas encore donnée.

Partie 2 — Application GRP Angular (Équipe Yassine)

Le projet de Yassine est un tableau de bord Angular dédié à la branche GRP (Gas, Renewables & Power) de Total Energies. Il centralise les KPI de 300 sites d'énergies renouvelables à travers le monde. J'ai dû monter rapidement en compétences sur JavaScript avancé et Angular pour contribuer directement au projet.

4

Ticket #1 — Filtrage dynamique au chargement

Afficher par défaut uniquement les KPI de l'année courante

Implémentation d'une logique de filtrage automatique dans le cycle de vie du composant Angular. Au lancement, le tableau n'affiche que les indicateurs de l'année en cours via manipulation des objets Date en JavaScript. Premier Pull Request validé par la Squad sur GitHub.

Angular TypeScript JavaScript GitHub PR
5

Ticket #2 — Export Excel

Exporter les données du tableau vers un fichier Excel téléchargeable

Bouton Export capturant l'objet de données du tableau filtré, conversion au format Excel, déclenchement automatique du téléchargement. Permet aux gestionnaires de réaliser leurs propres analyses statistiques et de partager des rapports de performance.

Angular JavaScript Export Excel UX
6

Ticket #3 — Bouton Import Excel

Mise à jour mensuelle massive du tableau de bord

Le ticket le plus complexe algorithmiquement. Sélection d'un fichier Excel local, parcours du fichier importé, comparaison ligne par ligne pour ne rajouter que les données nouvelles ou différentes (anti-doublons), injection dynamique dans le tableau sans rechargement. Une tâche de plusieurs heures manuellement accomplie en quelques secondes.

Angular TypeScript Algorithmique Import fichier Jira
Ce stage m'a fait passer du statut d'étudiant à celui de développeur capable de s'intégrer dans un projet réel de grande envergure. Apprendre Angular en autonomie, travailler en anglais quotidiennement avec des équipes indiennes, utiliser Jira et GitHub comme des outils de production — ce sont des compétences que seule une immersion en entreprise peut donner. Je repars avec une vision globale du cycle de vie d'un produit numérique, de la planification au déploiement.
