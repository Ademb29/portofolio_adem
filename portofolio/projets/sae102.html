Le projet en bref

À partir d'un scénario de réseau social, j'ai conçu une base de données complète avec utilisateurs, publications, commentaires, amis, likes. J'ai d'abord construit un modèle conceptuel (MCD), avant de traduire le tout en SQL pour créer les tables, insérer des données et écrire des requêtes. Ce projet m'a permis de comprendre en profondeur la modélisation de données et les liens entre différentes entités dans un système complexe.

Déroulement

1

Analyse du scénario

Identification des entités : Utilisateur, Publication, Commentaire, Amitié, Like. Définition des cardinalités.

2

Conception MCD → MLD

Modèle Conceptuel puis Logique. Normalisation des tables pour éviter les redondances.

3

Implémentation SQL

Création des tables, clés primaires et étrangères, contraintes d'intégrité, insertion de données de test.

4

Requêtes d'analyse

SELECT complexes avec jointures, GROUP BY, HAVING pour analyser les interactions du réseau social.

La SAÉ qui m'a le plus sensibilisé à l'importance de bien penser une architecture de données avant d'écrire une seule ligne de code. Un MCD mal conçu engendre des problèmes en cascade. J'ai compris pourquoi les bases de données relationnelles sont encore au cœur de presque tout système.
